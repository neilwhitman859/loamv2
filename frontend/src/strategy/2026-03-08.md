## What's Next

### Immediate Priorities

1. **Wine insights** — 110K wines at 0% enrichment. This is the biggest content gap. Strategy: batch by importance (vintage count as proxy), top 10K first. Estimated cost: $8-12 for Haiku.

2. **Producer insights** — 32K producers at 0%. Faster since there are fewer. Estimated cost: $3-5 for Haiku.

3. **More score sources** — Currently Vivino-only. Wine-Searcher, CellarTracker, and publication-specific scores would dramatically improve coverage beyond the current 23% of wines.

4. **Geographic enrichment** — Appellation lat/lng coordinates would unlock:
   - Map visualizations
   - Weather data (Open-Meteo is free)
   - Water body proximity calculations
   - Growing season validation

### Medium-Term

5. **Wine vintage insights** — Per-vintage AI analysis with weather context. Depends on having more scores + weather data.

6. **Soil & terroir data** — `appellation_soils`, `region_soils`, `wine_soils` tables are empty. AI inference from appellation names + producer websites.

7. **Regulatory data** — Min aging, max yield, allowed grapes, regulatory body for appellations. Web scraping + AI extraction.

### Strategic Questions

- **What's the target user experience?** Wine encyclopedia vs buying guide vs sommelier assistant. Each emphasizes different data.
- **Score data licensing** — Critic scores from Wine Spectator, Parker, etc. have licensing implications. CellarTracker community data may be more accessible.
- **Real-time vs batch enrichment** — Enrich everything upfront, or on-demand when first viewed?
- **Price data freshness** — Current prices are a snapshot. How often to refresh?

## Architecture Notes

### Tech Stack
- **Database**: Supabase PostgreSQL (project: `vgbppjhmvbggfjztzobl`)
- **Frontend**: Vite 7 + React 19 + TypeScript 5.9 + Tailwind CSS v4
- **Hosting**: Render static site with SPA rewrite
- **AI**: Claude Haiku for batch enrichment, ~$0.57 total spend to date

### Key Design Decisions
- UUID primary keys with unique slug constraints
- Explicit FK hints in Supabase queries (e.g., `countries!country_id`)
- Dedup indexes on score/price tables for idempotent ingestion
- Source tracking with `{field}_source` foreign keys
- Build timestamp injected via Vite `define` for deploy tracking
